# ADR-008 – Axios Choice

## Context
Define how the frontend will make HTTP requests.

## Decision
We will use **Axios**.

## Justification
- TypeScript support  
- Simpler error handling  
- Popular in the market

## Consequences
- ✅ Cleaner request code  
- ❌ External dependency (needs installation)

# ADR-008 – Escolha do Axios (PT)

## Contexto
Definir como o frontend fará requisições HTTP.

## Decisão

## Justificativa
- Suporte a TypeScript  
- Tratamento de erros mais simples  
- Popular no mercado
