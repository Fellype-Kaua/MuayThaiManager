# ADR-010 – Commit Convention

## Context
We needed to define a commit message convention to keep the Git history organized, consistent, and professional. This improves collaboration, makes the project evolution easier to read, and demonstrates good practices in the portfolio.

## Decision
We decided to adopt **Conventional Commits**, following the format:  
`<type>(scope): <short description>`

## Justification
- Keeps history clean and readable  
- Makes it easy to identify the type of change  
- Allows future automation for changelogs  
- Demonstrates maturity and organization in Git usage

## Consequences
- ✅ Clean, consistent, and professional commit history  
- ✅ Easier collaboration and maintenance  
- ❌ Requires discipline to always follow the convention

# ADR-010 – Convenção de Commits (PT)

## Contexto
Precisávamos definir uma convenção de mensagens de commit para manter o histórico do Git organizado, consistente e profissional. Isso melhora a colaboração, facilita a leitura da evolução do projeto e demonstra boas práticas no portfólio.

## Decisão
Optamos por adotar **Conventional Commits**, seguindo o formato:  
`<tipo>(escopo): <descrição curta>`

## Justificativa
- Mantém o histórico limpo e legível  
- Facilita identificar rapidamente o tipo de mudança  
- Permite automação futura de changelogs  
- Demonstra maturidade e organização no uso do Git

## Consequências
- ✅ Histórico limpo, consistente e profissional  
- ✅ Colaboração e manutenção facilitadas  
- ❌ Exige disciplina para seguir sempre o padrão
