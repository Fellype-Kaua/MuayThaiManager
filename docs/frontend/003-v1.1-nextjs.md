# ADR-003 – Next.js Choice (Updated)

## Context
Decide whether the frontend should use Next.js or React (Vite).
This ADR replaces the previous decision where Next.js was not adopted.

## Decision
We will use **Next.js (App Router)** instead of React + Vite.

## Justification
- Need for **SSR**, **SSG**, and improved **SEO**
- More secure authentication with **HTTP-only cookies**
- Route protection using **middleware** at the server boundary
- Cleaner architecture with **server components**
- Better long-term scalability and file-based routing
- Seamless integration with backend and cookie handling

## Consequences
✅ Stronger security and protected routes
✅ Better performance and SEO
❌ Increased initial complexity
❌ Requires migration of existing React code

# ADR-003 – Escolha pelo Next.js (Atualizada) (PT)

## Contexto
Decidir se o frontend deve usar Next.js ou React (Vite).
Esta ADR substitui a decisão anterior, que optava por não usar Next.js.

## Decisão
Usaremos **Next.js (App Router)** no frontend, em vez de React + Vite.

## Justificativa
- Necessidade de **SSR**, **SSG** e melhor **SEO**
- Autenticação mais segura utilizando **cookies HTTP-only**
- Proteção de rotas através de **middleware** antes da renderização
- Arquitetura mais limpa com **server components**
- Melhor escalabilidade e roteamento baseado em arquivos
- Integração mais consistente com o backend e fluxo de cookies

## Consequências
✅ Segurança reforçada e rotas protegidas
✅ Melhor performance inicial e SEO
❌ Complexidade maior em comparação ao Vite
❌ Exige migração do código já existente
